# Codex Loop

Dieser Ordner ist bewusst versioniert.

Er zeigt den letzten bekannten sichtbaren Laufstand des lokalen Autopilot-Loops:

- `issue.md`: aktuell bearbeitetes Issue
- `worker.md`: letzte Worker-Ausgabe
- `review.md`: letzte Reviewer-Ausgabe
- `doc-worker.md`: letzte Doku-Worker-Ausgabe

Änderungen in diesem Ordner blockieren den Loop-Start nicht.
Wenn ein Issue echte Projektänderungen erzeugt, wird `.codex-loop` zusammen mit Branch und PR mitgeführt.
Ein reiner `.codex-loop`-Diff zählt aber nicht als gelöstes Issue.
